Candidate thesis

A cross-domain issue is not closed when the immediate symptom disappears. It is closed when the system can explain the behavior: the violated invariant is named, the condition is reproducible, the authoritative configuration is known, the responsible decision is explicit, the repair is verified, and recurrence is harder.

System Coherence Frame

A practical closure model for autonomy integration

Use the frame to force agreement among behavior, architecture, evidence, configuration, and ownership. The specific tooling and thresholds would be learned from Latitude; the closure logic is deliberately system-agnostic.

Four closure moves
MoveQuestionDurable output
1. Find the fractureWhere does observed behavior first diverge from expected system behavior?Precise symptom, condition envelope, evidence gaps, and candidate boundaries
2. Establish the invariantWhich assumption must remain true across software, hardware, timing, data, calibration, or ownership?Testable invariant and minimum reproducible condition
3. Restore the couplingWhat code, configuration, interface, timing, calibration, or authority change makes the layers coherent?Repair with explicit decision owner and adjacent-risk review
4. Make recurrence harderWhere should evidence, validation, compatibility, or process now fail fast?Regression test, configuration check, observability, interface contract, or standard work

Integration case discussion

A useful interview format would be a sanitized integration scenario containing a symptom description, partial logs, platform/configuration identities, recent changes, and two plausible ownership boundaries. The discussion would make the operating method visible by asking the candidate to:

1. State assumptions and missing evidence.
2. Define the minimum reproducible condition.
3. Identify the likely violated invariant and competing hypotheses.
4. Design the next two experiments and expected discriminating evidence.
5. Define who should decide, who should contribute, and what blocks release.
6. Propose the regression or configuration mechanism that would make recurrence harder.
